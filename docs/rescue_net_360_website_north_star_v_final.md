# RescueNet360 Website North Star
## Static Trust & Routing Layer (Final)

---

# 1. Purpose

RescueNet360.com is a **static trust and routing website**.

It is NOT the product.
The product exists at: app.rescuenet360.com

---

## The Website Has Exactly Four Jobs

1. Get found via SEO
2. Build trust in under 10 seconds
3. Split audiences clearly
4. Hand off to the app with zero friction

Anything outside these four jobs does not belong on this site.

---

# 2. Core Positioning

A national pet identity system that helps reunite pets faster by connecting people, shelters, and data.

---

# 3. Audience Model

## Two Audiences Only

### Pet Owners
- Want safety and recovery

### Shelters & Rescues
- Want coordination and better outcomes

---

# 4. Brand & Design System

## Color Meaning
- Purple = identity, care, emotional connection
- Teal = system intelligence, coordination, signals

## Tone
Must feel:
- Calm
- Capable
- Clear
- Trustworthy

Must NOT feel:
- Charity guilt
- Emotional manipulation
- Corporate SaaS
- Veterinary clinic

---

## Flat vs Soft

### Flat
- Logo
- Icons
- Brand marks
- No gradients or shadows

### Soft
- UI surfaces
- Cards
- Sections
- Rounded corners
- Subtle shadows

---

## Anti-Patterns

Do NOT allow:
- Vet clinic look
- Emotional overload
- SaaS dashboard UI
- Hard borders
- Cluttered layouts

---

# 5. Navigation Model

## Guided Navigation (Minimal)

Header must include:
- Logo
- Optional descriptor
- Navigation links (max 4)
- Primary CTA: Start

### Allowed Links Only
- How It Works
- For Pet Owners
- For Shelters & Rescues
- About

Rules:
- No dropdowns
- No nested menus
- No expansion beyond this list

---

# 6. First-Time Entry Model

## Behavior

If NEW user:
- Show entry selection on homepage

If RETURNING user:
- Show normal homepage

---

## Entry Options

- Register Your Pet
- Apply as a Pilot Partner

---

## Rules

- Use localStorage or cookie
- Only show on homepage
- Never block direct page access
- Never force redirect

---

---

# 8. UX Rules

## Direct Entry

Every page must:
- Stand alone
- Explain system immediately
- Establish trust
- Include CTA

---

## No Scroll Dependency

User must be able to:
- Understand what this is
- Understand why it matters
- Take action

WITHOUT scrolling

---

## Page Structure

All pages must follow:

1. Header
2. Hero
3. Supporting Content
4. CTA

---

## CTA Rules

### Standardized CTAs

Pet Owner:
- Register Your Pet
- Report a Lost Pet (secondary)

Shelter:
- Apply as a Pilot Partner

Routing:
- Start

---

## CTA Placement

- One primary CTA above the fold
- One reinforcing CTA at bottom
- Max two total

---

# 9. Layered Content Model

Each page supports:

## Primary
- Immediate understanding
- Clear CTA

## Secondary
- Deeper explanation below fold

Rules:
- No new CTAs
- No new flows
- No change in page purpose

---

# 10. Trust System

Trust must come from:
- Clarity
- Consistency
- Simplicity

Not from persuasion.

---

# 11. Handoff Standard

The app must:
- Match color system
- Match logo
- Match tone

If transition is noticeable, system has failed.

---

## Technical Requirements

- Use full URLs
- No redirects
- Direct destination links

---

# 12. Performance

- Static-first
- Fast load times
- No heavy above-fold assets

Speed = trust

---

# 13. Guardrails

Developers must NOT:
- Add pages
- Add CTAs
- Expand scope
- Add navigation

This system is intentionally constrained.

---

# 14. Success Criteria

Each page must pass:

- User understands system in 5 seconds
- User knows what to do next
- Only one clear action exists

---

# 15. Final North Star Statement

The RescueNet360 website is a static trust and routing layer that attracts the right users, builds immediate credibility, and directs them into the product without friction, confusion, or loss of trust.
